Prompt 01 of 05
1
Dashboard Shell · Layout Architecture
Full Three-Panel Dashboard Layout
Establishes the complete application frame — sidebar, chat panel, live editor — with all layout rules, navigation architecture, and global design language baked in from the first generation
Foundation Run First ~340 words
Final Figma Make Prompt — copy exactly as written
Visual Outcome
Create a production-ready SaaS web application dashboard for ConvoCrystal, an AI-powered meeting transcript analysis platform. The final output is a fully realized three-panel desktop interface showing a populated, active state: the left sidebar contains a navigation hierarchy and file library, the center panel shows an active LLM chat session with streaming responses, and the right panel shows a live rich-text document being edited. This is the master application shell — every design decision made here cascades to all sub-components.
Brand Identity
Apply a dark, technical-minimal aesthetic throughout. Base background: near-black deep navy (#0B0C10). Panel surfaces: slightly lighter (#1A1D2E). Primary accent: electric indigo-blue (#6366F1) used sparingly on interactive elements, active states, and focus rings only. Secondary accent: ice teal (#06B6D4) on streaming indicators and live states. All borders are 1px at low opacity (#2A2D42). Typography: Cabinet Grotesk ExtraBold for headings, DM Sans for body and UI labels, JetBrains Mono for timestamps, token counts, and code-adjacent metadata. Text hierarchy: primary (#E8EAF6), secondary (#9BA3C8), tertiary/muted (#5C6490). The overall palette reads as a precision instrument — deep, focused, zero decoration noise. No gradients on backgrounds. Subtle 1px grid lines may be used as structural dividers. The only glow effect allowed is a soft radial behind the primary CTA button and active sidebar items (3px radius, 15% opacity indigo).
Interaction Behaviors
Sidebar items: on hover, show a left-side 2px indigo border reveal with a background tint transition (160ms ease-out). Active state: filled indigo background at 15% opacity, white text, left border solid. Collapsed sidebar: icons only at 56px width with tooltip on hover. Top navigation bar: 48px fixed height, brand wordmark left-aligned, user avatar + workspace switcher right-aligned. Panel resize handles: show on hover at panel boundaries, draggable with 200ms spring animation. The entire layout must feel engineered — transitions are never decorative, always purposeful and sub-200ms.
Responsive Adaptation
Desktop (1280px+): Three-panel layout. Left sidebar 240px fixed. Center and right panels share remaining space at approximately 1:1.4 ratio, both resizable. Tablet (768–1023px): Two panels — sidebar collapses to icon rail, center expands. Right editor panel slides in as overlay drawer (360px) triggered by document icon. Mobile (<768px): Single panel with bottom tab bar (4 tabs: Library, Chat, Editor, Settings). Panel switching is a horizontal slide animation (320ms cubic-bezier decelerate). No panel should be visible simultaneously below 1024px.
Structural Sections Required
Left Sidebar: ConvoCrystal brand mark at top, collapsible nav sections (Library, Workspaces, Recent, Settings), transcript file list with confidence score badges, user avatar + plan indicator at bottom. Center Panel: Topbar with transcript context label and session controls, message thread with both user and AI bubble variants, processing stage indicator strip above the latest AI message (Extracting / Analyzing / Generating steps), input composer at bottom with send button and interrupt button. Right Panel: Document title field, formatting toolbar (bold, italic, headings, lists, AI Refine button highlighted in indigo), rich text editing area with section highlights, export dropdown (PDF, DOCX, Notion, Markdown), version commit button in toolbar. Show the interface in a populated, mid-session state — not empty, not loading.
① Outcome: three-panel dashboard ② Brand: dark navy + electric indigo ③ Interaction: hover + resize + collapse ④ Responsive: 3 breakpoints defined ⑤ Sections: sidebar + chat + editor
Words: ~340 Sections: 5 Scope: Full dashboard Run order: First
Prompt 02 of 05

Upload Flow · Progress States · Format Detection
Multi-Step Transcript Upload Flow
Generates the complete file upload experience across all five states: idle drop zone, validation, uploading with progress, processing with stage indicators, and completion — including error and empty states
Module Run Second ~320 words
Final Figma Make Prompt — copy exactly as written
Visual Outcome
Create a multi-step file upload modal and embedded upload flow panel for ConvoCrystal, an AI transcript analysis SaaS. The output is a complete UI showing all five upload states as separate screens or artboards: (1) Idle drop zone, (2) File selected with validation feedback, (3) Active upload with live progress, (4) Backend processing with stage tracker, (5) Upload complete with first-analysis preview. This component appears both as a full-page onboarding flow for new users and as an inline panel within the existing three-panel dashboard layout.
Brand Identity
Maintain the ConvoCrystal dark technical aesthetic: deep navy surfaces (#0B0C10 base, #1A1D2E panels), electric indigo (#6366F1) as the primary active accent, ice teal (#06B6D4) on the processing stage indicator's active step, DM Sans for all copy, JetBrains Mono for file size, byte counts, percentage values, SHA hashes, and format badges (VTT, SRT, TXT, DOCX, JSON). The drop zone should feel like a clean terminal input area — not playful or rounded. Use a dashed 1px border (2px dash, 4px gap pattern) that becomes solid indigo on drag-over. Format detection chips (the file type labels) are small monospaced pill badges in ice teal on hover.
Interaction Behaviors
Idle state: Dashed border animates with a slow rotating dash-offset (8s loop, subtle). On drag-over: border becomes solid indigo, background tints to 4% indigo, drop zone icon scales 1.08x with spring ease. File selected: Card slides in from below (200ms decelerate) showing filename, format badge, and file size. Validation phases animate sequentially — each check marks done with a checkmark before the next begins. Upload progress: SVG ring progress indicator with smooth stroke-dashoffset animation, percentage in JetBrains Mono at center. Processing stages: Vertical timeline, active step pulses with a 1.5s ease-in-out opacity animation. Completed stages show a filled checkmark in emerald green (#10B981).
Responsive Adaptation
Desktop: Upload component appears as a centered modal overlay (640px max-width) or as a right-drawer panel within the sidebar. The five state screens can be presented as a horizontal scroll carousel on desktop for design reference. Tablet: Modal adapts to 90% viewport width, stage tracker switches from horizontal to vertical layout below 640px. Mobile: Full-screen overlay with bottom sheet for file metadata. The drop zone becomes a tap-to-browse button only (no drag on mobile — the drag zone is hidden entirely). Progress ring shrinks to 64px diameter. Stage tracker text truncates to single-word labels.
Structural Sections Required
State 1 — Drop Zone: Large centered dashed container with cloud-upload icon, headline ("Drop your transcript here"), accepted format chips, "or browse files" secondary link, and "Try a sample transcript" ghost CTA. State 2 — Validation: File preview card (icon, name, size, detected format badge), three-phase validation ticker (Format Check ✓, Size Check ✓, Content Scan…), cancel button. State 3 — Upload Progress: Circular progress ring with byte counter and estimated time, file name header, cancel/pause button row, progress percentage. State 4 — Processing Stages: Vertical 5-step timeline (Queued → Parsing → Extracting → Analyzing → Scoring) with active step highlighted in indigo, elapsed time counter, animated connecting lines. State 5 — Complete: Success checkmark animation, transcript title + duration + speaker count metadata, "View Analysis" primary CTA in indigo, "Upload Another" ghost CTA. Error state: Red-bordered card with error type, explanation, and retry or contact-support options.
① Outcome: 5-state upload flow ② Brand: mono badges + dashed zones ③ Interaction: drag + ring progress + stage pulse ④ Responsive: modal + full-screen mobile ⑤ Sections: 5 states + error state
Words: ~320 States: 5 + error Scope: Upload module Run order: Second
Prompt 03 of 05
3
Chat UI · Token Streaming · Stage Indicators · Confidence Scoring
Real-Time LLM Chat Interface
Designs the complete chat panel including the streaming message experience, LangGraph orchestration stage bar, RAGAS confidence scoring display, citation links, suggested prompt chips, and the full input composer
Module Run Third ~350 words
Final Figma Make Prompt — copy exactly as written
Visual Outcome
Create the complete LLM chat panel for ConvoCrystal, showing a realistic mid-conversation state with three to five message pairs visible, one AI message currently streaming (partially generated text with a blinking cursor), and a processing stage indicator bar actively showing the "Generating" step. The panel must convey the feeling of a live, intelligent system mid-thought — not a static chat mockup. Include the full message thread, an active input composer, a chip row of suggested prompts, and a confidence score indicator attached to the most recent completed AI response. This is the center panel of the three-panel layout.
Brand Identity
Chat interface inherits ConvoCrystal's dark technical palette. User message bubbles: right-aligned, indigo (#6366F1) filled background, white DM Sans text, 12px border radius, maximum 72% panel width. AI message bubbles: left-aligned, panel surface (#1A1D2E) background, 1px indigo border on active/streaming messages (transitions off when complete), text in #E8EAF6. Streaming AI message has a 2px left border in ice teal (#06B6D4) that disappears on completion. The blinking cursor is a 2px × 14px indigo rectangle, 1s blink cycle. Stage indicator bar: compact 28px tall strip above the streaming message, showing icons and labels for each orchestration step (Extracting context, Analyzing semantics, Generating response) with the active step in indigo and completed steps in emerald. Confidence score badge: small pill below completed messages showing the RAGAS composite score (0–100%), color-coded: ≥85% emerald, 65–84% amber, <65% rose.
Interaction Behaviors
Streaming animation: New tokens appear with a micro-fade from opacity 0.4 to 1 over 60ms with a 1px upward translate — giving the feeling of words materializing rather than appearing. The text container grows smoothly as content accumulates (no jumpy height shifts). Auto-scroll: the message list follows the bottom of streaming content smoothly; a "↑ Scroll to read" chip appears if the user manually scrolls up during streaming. Hover on completed AI messages: reveals three action icons (copy, push to editor, cite) in the message's top-right corner, fading in at 150ms. Input composer: Expands vertically from 1 line to 4 lines max as content grows. Send button is indigo, switches to an animated "stop" square during active streaming. Suggested prompt chips: horizontal scrollable rail, hover state shifts border to indigo and text brightens.
Responsive Adaptation
Desktop: Chat panel is the center column, fixed width bounded by sidebar and editor panel. No horizontal scroll. Stage indicator collapses to icon-only with tooltip labels below 960px panel width. Tablet: Chat panel expands to fill most of the viewport when editor is hidden. Suggested prompts collapse to a "Suggestions ▾" expandable button. Mobile: Full-screen chat view, input composer fixed to bottom of viewport, message list scrolls above it. Stage indicator is a single-line compact bar with abbreviated stage name. Bottom safe area padding for devices with home bar.
Structural Sections Required
Chat Header: Transcript context label (filename + duration), connection status dot (green = connected), session controls (new session, clear, export chat). Message Thread: Three completed exchange pairs showing realistic questions and structured AI responses. One streaming message in progress — show approximately 40% of a response generated with cursor active. Stage Indicator Bar: Compact three-step horizontal strip (Extracting ✓ / Analyzing ✓ / Generating →) directly above the streaming bubble. Confidence Panel: Below a completed AI message, a row showing: overall score pill (e.g. 91%), and three sub-scores (Faithfulness 94%, Relevance 88%, Precision 92%) each with a small bar indicator. Citation Links: Numbered reference chips below AI message text linking to transcript segments (e.g. [1] Speaker A · 14:32). Suggested Prompts Rail: Horizontal scrollable chip row with 4–5 context-aware questions. Input Composer: Multi-line textarea with send and interrupt buttons, character/token counter in mono, context-pin toggle button.
① Outcome: live mid-session chat state ② Brand: bubble variants + stage colors ③ Interaction: token fade + autoscroll + hover actions ④ Responsive: panel + full-screen + mobile ⑤ Sections: header + thread + stage + confidence
Words: ~350 States: streaming + complete + empty Scope: Chat module Run order: Third

Live Document Editor with AI Refinement
Generates the right-panel rich text editor with section highlighting, the floating LLM refinement popover, inline streaming token insertion, accept/reject preview UI, and the complete export workflow

Final Figma Make Prompt — copy exactly as written
Visual Outcome
Create the live document editor panel for ConvoCrystal, shown in an active mid-editing state. The document contains four to six structured sections generated from a meeting transcript (Summary, Key Decisions, Action Items, Open Questions, Risks Flagged). One section of text is currently selected and the AI Refinement popover is open above it, showing refinement action options. A second section shows an inline streaming state — AI-generated replacement text materializing character-by-character with a blinking cursor. A third section shows the accept/reject preview banner. The editor is the right panel of the three-panel dashboard layout and shares the same dark design system.
Brand Identity
Editor background: same surface (#1A1D2E) as other panels. Document content area has a subtle inset inner shadow (1px, 10% opacity) to distinguish it from the panel chrome. Toolbar: 48px height strip, icon buttons at 32×32px with 6px radius. Active toolbar button: filled indigo background, white icon. The AI Refine button in the toolbar is always highlighted with a 1px indigo border and indigo icon — it is the primary editor CTA. Selected text: background highlight in indigo at 18% opacity. Section with inline streaming: left border 2px indigo, background at 5% indigo opacity, original text dimmed to 35% opacity while replacement streams in. AI-generated text mark: subtle underline in indigo at 30% opacity to indicate provenance. Section labels (Summary, Decisions) in JetBrains Mono uppercase 10px, letter-spacing 0.15em, color: muted (#5C6490).
Interaction Behaviors
Text selection: The refinement popover appears 8px above the selection midpoint, fading in at 120ms with a slight upward translate. Popover contains six action buttons: Refine, Expand, Shorten, Formalize, Simplify, Rephrase — each 28px tall with hover fill in indigo at 10%. Inline streaming: The selected text fades to 35% opacity simultaneously as the replacement text streams in character-by-character to its right in a separate text run. Streaming cursor: same 2px rectangle blink as in chat. Accept/Reject: A compact two-button banner (emerald Accept, rose Reject) pins to the bottom edge of the streaming section. On accept, the dimmed original text cross-dissolves out and replacement snaps to full opacity at 200ms. On reject, original text snaps back to full opacity, replacement fades out. Commit button in toolbar triggers a modal with commit message input and branch selector.
Responsive Adaptation
Desktop: Right panel, fixed position, minimum width 320px, resizable. Toolbar stays horizontal at all widths. Below 360px panel width, toolbar wraps into two rows. Tablet: Editor appears as a slide-in drawer from the right side (360px wide), triggered by a document icon in the center panel header. Drawer has its own close button and overlay backdrop. Refinement popover is repositioned to bottom of selection (not above) to avoid viewport clipping. Mobile: Editor is a full-screen tab accessed from the bottom tab bar. Toolbar scrolls horizontally. The refinement popover becomes a bottom sheet with the same six action options displayed as full-width buttons (44px each touch target).
Structural Sections Required
Editor Toolbar: Document title (editable inline), separator, format buttons (B, I, U, H1, H2, H3, UL, OL, Quote), separator, AI Refine button (highlighted), separator, Commit button, Export menu button. Editor Content Area: Five structured sections with section-type labels, realistic meeting intelligence content. Section highlight states: normal, selected (highlighted), streaming (active), and committed (with AI mark underline). Refinement Popover: Floating card with six action buttons and a close X. Inline Streaming Section: Show the before-state (dimmed original) and during-state (streaming replacement) simultaneously for design reference. Accept/Reject Banner: Two-button row pinned to streaming section bottom edge. Collaborator Presence Bar: Three avatar chips in the top-right of the editor showing live cursor positions (different colors per user). Export Dropdown: PDF, DOCX, Notion push, Markdown copy, Slack post — each with a platform icon.
① Outcome: mid-edit active editor panel ② Brand: selection highlight + mono labels ③ Interaction: popover + streaming + accept/reject ④ Responsive: panel + drawer + full-screen ⑤ Sections: toolbar + content + refinement + export
Words: ~335 States: normal + streaming + preview Scope: Editor module Run order: Fourth

Version Timeline · Diff View · Auth Lifecycle · Onboarding
Version Control Timeline + Full Auth Lifecycle
Generates the version control timeline panel with branching visualization, side-by-side diff comparison, and restore flow — plus the complete authentication suite: login, registration, password recovery, and onboarding screens
Module Run Fifth ~360 words
Final Figma Make Prompt — copy exactly as written
Visual Outcome
Create two distinct UI surface groups for ConvoCrystal: (A) The version control timeline panel, shown as a full-screen modal overlaying the dashboard — displaying a Git-style commit history graph with branches, a selected commit's diff comparison view (side-by-side), and a restore flow confirmation screen. (B) The complete authentication lifecycle as separate full-page screens: Login, Register (3-step), Email Verification holding screen, and Password Recovery (request + reset). Both groups share the ConvoCrystal design system but serve very different visual moods — the version timeline is dense and data-rich; the auth screens are spacious and focused.
Brand Identity
Version timeline: same dark panel surfaces as the main app. Commit nodes on the graph are 12px circles — filled indigo (#6366F1) for HEAD, filled surface (#2A2D42) for older commits, filled violet (#8B5CF6) for branch points. Branch lines are 1px SVG paths in matching colors. Diff view: removed lines have a subtle rose (#F43F5E) background tint at 10%; added lines have emerald (#10B981) at 10%. Changed characters within a line are highlighted at 20% opacity. Line numbers in JetBrains Mono 11px, muted color. Auth screens: invert the palette — use a light background (#F7F6F3 warm off-white) with the ConvoCrystal wordmark centered at top in dark ink. Input fields: white background, 1px border (#D0D0E0), 4px radius, indigo (#6366F1) focus ring (3px offset). Primary button: solid indigo fill, white DM Sans 600 text. OAuth buttons (Google, GitHub): white background, dark border, icon + label. The auth screens feel like the light-mode counterpart of the dark dashboard — same brand DNA, different energy.
Interaction Behaviors
Version timeline scrubbing: hovering over a commit node highlights the connected branch lines and shows a tooltip card with commit metadata (SHA hash, author, message, timestamp, diff stats). Clicking a node selects it and slides the diff panel in from the right at 280ms ease-out. The restore button in the diff panel triggers a confirmation dialog with a warning about creating an automatic backup branch before proceeding. Diff view: a toggle button switches between "Split" and "Unified" view — the panel cross-fades between the two layouts at 200ms. Auth forms: Input validation shows inline beneath each field on blur (not on change). Submit button disables and shows a small spinner during submission. Rate-limited state shows a countdown timer beneath the submit button. OAuth flow: clicking an OAuth button fades the form out and shows a loading state while the redirect occurs.
Responsive Adaptation
Version timeline modal: Full-screen overlay on all breakpoints. On desktop: two-column layout (commit list left, diff panel right). On tablet: commit list collapses to a slim timeline strip, diff takes most of the space. On mobile: commit list is a bottom drawer that expands up; diff view is single-column unified mode only (side-by-side is hidden on mobile). Auth screens: all auth pages are single-column centered on all breakpoints. Maximum content width 480px. On mobile, the ConvoCrystal wordmark is smaller and the form fills 90% viewport width. Keyboard-aware layout: form shifts up when virtual keyboard appears on iOS/Android.
Structural Sections Required
Version Timeline Modal: Header with document title + branch selector dropdown + Close button. Left column: Branch selector tabs, vertical commit list with SVG branch graph (show one main branch and one feature branch diverging and merging), each commit item showing: abbreviated SHA hash chip (JetBrains Mono), commit message, author avatar + name, relative timestamp, diff stats (+N -N). Right column: Diff header (comparing commit A to commit B), Toggle split/unified button, File path label, Side-by-side diff with line numbers and change highlighting, Restore button (indigo), "Restoring will auto-backup current state to a new branch" warning note. Restore Confirmation Dialog: Warning icon, message explaining the backup branch that will be created, input for commit message, Confirm (indigo) and Cancel buttons. Auth — Login Screen: Wordmark, H1 "Welcome back", email + password fields, "Forgot password?" link, Sign In button, divider "or continue with", Google + GitHub OAuth buttons, "Don't have an account? Register" link. Auth — Register Screen: Wordmark, 3-step stepper indicator, Step 1 fields (name, email, password with strength indicator), Step 2 email verification holding state (envelope icon, check email message, resend link with 60s cooldown), Step 3 workspace setup (workspace name input, invite teammates optional). Auth — Password Recovery: Request screen (email field, "Send reset link" button, generic success message). Reset screen (new password + confirm fields, strength meter, "Set new password" button).
① Outcome: version timeline + auth suite ② Brand: dark timeline + light auth dual-palette ③ Interaction: scrub + diff toggle + form validation ④ Responsive: modal + bottom drawer + single-col auth ⑤ Sections: timeline + diff + restore + login + register + recovery
Words: ~360 Screens: 7 distinct screens Scope: Versions + Auth Run order: Fifth
